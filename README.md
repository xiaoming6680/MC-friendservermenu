# friendservermenu

`friendservermenu` 是一个面向 Minecraft 朋友服的轻量 Fabric MOD。它提供一个真正的客户端自定义 GUI，让玩家通过 `/menu` 在游戏内完成公共传送点、坐标分享、玩家任务、活动通知、服务器状态查看和 OP 管理等常用操作。

MOD ID：`friendservermenu`

## 适用版本

- Minecraft Java Edition `1.21.11`
- Fabric Loader `0.19.2` 或更新版本
- Fabric API `0.141.4+1.21.11`
- Java 21

客户端和服务端都需要安装本 MOD。客户端负责 GUI、输入框和 HUD 渲染；服务端负责权限检查、传送、任务、活动、奖励和配置保存。

## 功能概览

- 自定义菜单：玩家输入 `/menu` 打开 GUI，并默认回到上次退出时所在的主界面。
- 首次初始化：服务器第一次使用时，OP 打开菜单会进入初始化界面，设置左上角显示的 GUI 名称。
- 公共传送点：玩家可新增、编辑公共传送点；创建者和 OP 可删除传送点。
- 坐标分享：支持复制坐标、公开坐标、私发坐标；公开坐标聊天消息附带 `[点我传送]`。
- 坐标 HUD：可在屏幕上显示当前维度、中文群系和坐标，支持开关、拖动、自由调整宽高比例。
- 玩家任务：玩家可发布公开或私人任务，其他玩家可加入、退出、邀请、编辑和确认完成。
- 任务 HUD：已加入任务可显示到 HUD，支持独立开关、拖动、自由调整宽高比例。
- 历史任务：参与过的已完成或已结束任务会保存在历史任务里，可进入详情查看。
- OP 奖励箱：OP 可把提前准备好的奖励放入任务奖励箱，任务完成后发给参与玩家。
- 完成庆祝：任务完成时在参与玩家身边播放无伤害烟花粒子、轻量音效，并给予少量经验。
- 活动通知：OP 可发布集合、发物品、维护、探索等活动模板。
- 维护倒计时：维护时间和维护倒计时输入框会互相同步，倒计时结束可执行停服。
- 服主管理：提供时间、天气、玩家、清理、重载配置等常用 OP 操作。
- 服务端校验：传送、奖励、管理操作和任务状态均由服务端校验，客户端 payload 不被信任。

## 安装

服务端：

1. 安装 Minecraft `1.21.11` 对应的 Fabric 服务端。
2. 将 Fabric API 放入服务端 `mods` 文件夹。
3. 将 `friendservermenu-1.0.0.jar` 放入服务端 `mods` 文件夹。
4. 启动服务端。

客户端：

1. 安装 Minecraft `1.21.11` 对应的 Fabric 客户端。
2. 将 Fabric API 放入客户端 `mods` 文件夹。
3. 将同一个 `friendservermenu-1.0.0.jar` 放入客户端 `mods` 文件夹。

## 首次使用

第一次进入世界或服务器时，如果 `config/friendservermenu.json` 还没有完成初始化，OP 玩家打开 `/menu` 会看到初始化页面。

初始化页面会要求输入 GUI 名称，例如：

```text
小铭的服务器菜单
```

点击完成后，名称会保存到配置文件，之后打开菜单时显示在 GUI 左上角。

如果需要重新测试初始化页面：

1. 关闭服务器或退出世界。
2. 打开运行目录下的 `config/friendservermenu.json`。
3. 删除该文件，或把 `"initialized": true` 改成 `"initialized": false`。
4. 重新进入服务器，确保自己是 OP。
5. 输入 `/menu`。

## 命令

| 命令 | 说明 |
|------|------|
| `/menu` | 打开 friendservermenu，并恢复上次退出时所在的主界面 |
| `/menu tasks` | 直接打开任务分页 |
| `/menu activity` | 直接打开活动分页 |
| `/adminmenu` | 直接打开服主管理分页，仅 OP 等级 2 及以上可用 |
| `/fsm_coord_tp <id>` | 点击公开坐标聊天按钮时使用的临时传送命令 |
| `/fsm_task_join <id>` | 点击任务邀请 `[加入任务]` 时使用的加入命令 |
| `/fsm_activity_claim <id>` | 点击活动 `[手动领取]` 时使用的领取命令 |

## 配置文件

主要服务端配置：

```text
config/friendservermenu.json
```

默认结构：

```json
{
  "menuTitle": "小铭的服务器菜单",
  "initialized": false,
  "locations": []
}
```

任务持久化文件：

```text
config/friendservermenu-tasks.json
```

客户端本地 HUD 设置：

```text
config/friendservermenu-hud.json
config/friendservermenu-coordinate-hud.json
```

`friendservermenu.json` 可以由 OP 在 GUI 中新增传送点，也可以手动编辑后在服主管理页点击“重载配置”生效。任务文件由服务端自动维护，不建议手动修改。

## 公共传送点

玩家可以在传送页点击“新增传送点”。普通玩家新增时不需要填写 ID、yaw 或 pitch，界面会保持简洁；服务端会根据名称自动生成不重复 ID。

权限规则：

- 普通玩家可以新增公共传送点。
- 普通玩家可以编辑公共传送点，但不能修改传送点 ID。
- 创建传送点的玩家可以删除自己创建的传送点。
- OP 可以删除任意传送点。
- 旧配置中没有创建者记录的传送点仍只能由 OP 删除。

服务端会校验维度、坐标、ID 重复和配置保存结果。

## 坐标与坐标 HUD

坐标页支持：

- 复制当前坐标。
- 向聊天栏公开坐标，并附带 `[点我传送]`。
- 只把坐标私发给自己。
- 显示当前中文群系。
- 开关坐标 HUD。
- 进入独立界面编辑坐标 HUD 位置和比例。

坐标 HUD 会显示当前维度、中文群系和坐标。玩家可以拖动位置，也可以通过 `变窄`、`变宽`、`变矮`、`变高` 自由调整 HUD 比例。

公开坐标的传送按钮不会把可信坐标交给客户端。服务端会保存一个短期有效的坐标分享 ID，按钮只执行：

```text
/fsm_coord_tp <id>
```

## 玩家任务

玩家可以在任务页发布任务，例如：

```text
完成刷怪塔
```

任务支持公开和私人两种可见性：

- 公开任务：所有玩家可见并可加入。
- 私人任务：发布者、成员、受邀玩家和 OP 可见。

任务支持：

- 加入和退出任务。
- 编辑任务标题、说明和可见性。
- 点击任务卡片进入详情页查看状态、说明、奖励和已加入玩家。
- 邀请在线玩家加入任务，受邀玩家会收到聊天提示和绿色 `[加入任务]`。
- 点击“已完成”参与完成确认。
- 当前任务 50% 成员点击“已完成”后，任务完成。
- 发布者可以结束任务。
- 历史任务会保留自己参与过的已完成或已结束任务。

任务会保存到：

```text
config/friendservermenu-tasks.json
```

服务器正常重启后，进行中任务和历史任务会继续保留。

## 任务奖励

OP 可以在任务卡片点击“奖励”按钮，打开 27 格箱子式奖励界面，把背包中提前准备好的物品放入任务奖励箱。

任务完成时：

- 在线参与者会立即获得奖励箱中的物品副本。
- 离线参与者下次进服时补发。
- 背包放不下的物品会掉落在玩家身边。
- 玩家会获得少量经验。
- 身边会播放无伤害烟花粒子和较小音量的庆祝音效。

已完成或已结束任务不能继续编辑奖励。

## 任务 HUD

任务 HUD 默认开启，只显示玩家已加入且仍在进行中的任务。

玩家可以：

- 在任务卡片点击“显示到HUD”。
- 在任务页开关 HUD。
- 进入“编辑HUD位置”独立界面拖动 HUD。
- 使用 `变窄`、`变宽`、`变矮`、`变高` 自由调整 HUD 比例。

玩家退出任务后，任务 HUD 会实时更新。

## 活动通知

OP 可以在活动页发布活动通知。当前模板包括：

- 集合活动：带集合地点和传送按钮。
- 发物品：玩家需要在聊天栏点击 `[手动领取]`，或在活动 GUI 点击“领取物品”。
- 维护通知：支持维护时间和维护倒计时，两个输入框会互相同步。
- 探索/副本：用于多人探索或副本集合。

维护倒计时会在活动页实时显示。服务端会在 `60/30/15/10/5/4/3/2/1` 秒提醒，并在倒计时结束时执行 `/stop`。

活动发布时离线的玩家加入服务器后会收到一次活动聊天提示，不会反复刷屏。

## 服主管理

OP 等级 2 及以上可以使用服主管理页：

- 时间管理：白天、正午、夜晚、午夜。
- 天气管理：晴天、下雨、雷暴。
- 玩家管理：传送、游戏模式、临时飞行、恢复生命、补饥饿、清效果、熄灭、Kick、Ban、授予或撤销 OP。
- 清理管理：清理掉落物、经验球、箭矢类投射物、敌对生物。
- 重载配置：重新读取 `friendservermenu.json`。

普通玩家即使伪造客户端 payload 请求管理员 action 或活动组织 action，服务端也会拒绝。

## 构建

Windows PowerShell：

```powershell
.\gradlew.bat build
```

构建完成后，产物位于：

```text
build/libs/friendservermenu-1.0.0.jar
build/libs/friendservermenu-1.0.0-sources.jar
```

## 项目结构

```text
src/main/java/       服务端与通用逻辑
src/client/java/     客户端 GUI、HUD、网络接收
src/main/resources/  fabric.mod.json、声音资源
```

## 不包含的功能

`friendservermenu` 不是领地、经济、商店、权限组、反作弊、数据库或网页后台 MOD。它的目标是给朋友服提供一个轻量、直观、由服务端校验的游戏内菜单。

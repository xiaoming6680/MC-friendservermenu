# XM Server Menu

推荐 GitHub 仓库名：`xm-server-menu`

`XM Server Menu` 是一个面向 Minecraft 朋友服的小型 Fabric MOD。它在游戏内提供“小铭的服务器菜单”，让玩家通过 `/menu` 打开自定义 GUI，完成公共传送点、坐标分享、玩家任务、活动通知、服务器状态查看和 OP 管理等常用操作。

当前 MOD ID 仍为 `friendservermenu`，构建产物名为 `friendservermenu-1.0.0.jar`。

## 适用环境

- Minecraft Java Edition `1.21.11`
- Fabric Loader `0.19.2` 或更新版本
- Fabric API `0.141.4+1.21.11`
- Java 21

客户端和服务端都需要安装本 MOD。客户端负责 GUI 和 HUD，服务端负责权限检查、传送、任务、活动和配置写入。

## 功能概览

- 自定义服务器菜单：使用 `/menu` 打开，普通打开会回到上次退出菜单时所在的主界面。
- 公共传送点：玩家可新增和编辑传送点，创建者和 OP 可删除。
- 坐标分享：复制坐标、公开坐标、私发坐标；公开坐标聊天消息附带 `[点我传送]`。
- 玩家任务：玩家可发布公开或私人任务，加入、退出、编辑、邀请成员、完成确认和查看历史任务。
- 任务 HUD：玩家可选择任务显示到 HUD，支持独立编辑位置和等比缩放。
- OP 奖励箱：OP 可给任务放入奖励，任务完成时发给参与玩家，离线成员下次进服补发。
- 活动通知：OP 可发布集合、发物品、维护、探索等活动模板。
- 维护倒计时：维护时间和维护倒计时输入框会互相同步。
- 服主管理：时间、天气、玩家、清理、重载配置等常用管理操作。
- 实时刷新：GUI 约每 1 秒同步一次服务端数据和状态。

## 安装

服务端：

1. 安装 Minecraft `1.21.11` 对应的 Fabric 服务端。
2. 将 Fabric API 放入服务端 `mods` 文件夹。
3. 将 `friendservermenu-1.0.0.jar` 放入服务端 `mods` 文件夹。
4. 启动服务端，首次运行会生成 `config/friendservermenu.json`。

客户端：

1. 安装 Minecraft `1.21.11` 对应的 Fabric 客户端。
2. 将 Fabric API 放入客户端 `mods` 文件夹。
3. 将同一个 `friendservermenu-1.0.0.jar` 放入客户端 `mods` 文件夹。

## 命令

| 命令 | 说明 |
|------|------|
| `/menu` | 打开小铭的服务器菜单，并恢复上次退出时所在的主界面 |
| `/menu tasks` | 直接打开任务分页 |
| `/menu activity` | 直接打开活动分页 |
| `/adminmenu` | 直接打开服主管理分页，仅 OP 等级 2 及以上可用 |
| `/fsm_coord_tp <id>` | 点击公开坐标聊天按钮时使用的临时传送命令 |
| `/fsm_task_join <id>` | 点击任务邀请 `[加入任务]` 时使用的加入命令 |
| `/fsm_activity_claim <id>` | 点击活动 `[手动领取]` 时使用的领取命令 |

## 配置

配置文件位置：

```text
config/friendservermenu.json
```

默认配置：

```json
{
  "menuTitle": "小铭的服务器菜单",
  "locations": []
}
```

不会自动生成示例传送点。传送点可以通过 GUI 新增，也可以由 OP 直接编辑配置文件后在菜单中点击“重载配置”生效。

## 公共传送点

玩家可以在传送页点击“新增传送点”。普通玩家新增时不需要填写 ID、yaw 或 pitch，服务端会根据名称自动生成不重复 ID。中文名称会生成 `tp_<hash>` 形式的 ID，避免退化成固定 ID。

权限规则：

- 普通玩家可以新增公共传送点。
- 普通玩家可以编辑公共传送点，但不能修改传送点 ID。
- 创建传送点的玩家可以删除自己创建的传送点。
- OP 可以删除任意传送点。
- 旧配置中没有创建者记录的传送点仍只能由 OP 删除。

服务端会校验维度、坐标、ID 重复和配置保存结果；客户端提交的数据不会被直接信任。

## 玩家任务

玩家可以在任务页发布任务，例如“完成刷怪塔”。任务支持公开和私人两种可见性：

- 公开任务：所有玩家可见并可加入。
- 私人任务：发布者、成员、受邀玩家和 OP 可见。

任务支持：

- 加入和退出任务。
- 编辑任务标题、说明和可见性。
- 点击任务卡片进入详情页查看说明、奖励、状态和已加入玩家。
- 邀请在线玩家加入任务，受邀玩家会收到聊天提示和绿色 `[加入任务]`。
- 点击“已完成”发起或参与完成确认。
- 当前任务成员数达到 50% 完成确认后，任务完成。
- 发布者可以结束任务。
- 历史任务会显示自己参与过的已完成或已结束任务，并可从自己的历史列表中删除。

任务 HUD 默认开启，只显示玩家已加入且仍在进行中的任务。玩家可以在任务页切换 HUD 显示，进入“编辑HUD位置”后拖动 HUD，并用滚轮或按钮等比调整大小。

任务系统目前是服务端运行期内存数据，服务器重启后不会保留任务、历史任务或任务奖励箱。

## 任务奖励

OP 可以在任务卡片点击“奖励”按钮，打开 27 格箱子式奖励界面，把背包中提前准备好的物品放入任务奖励箱。

任务完成时：

- 在线参与者会立即获得奖励箱中的物品副本。
- 离线参与者下次进服时补发。
- 背包放不下的物品会掉落在玩家身边。
- 已完成或已结束任务不能继续编辑奖励。

## 活动通知

OP 可以在活动页发布活动通知。当前模板包括：

- 集合活动：带集合地点和传送按钮。
- 发物品：玩家需要在聊天栏点击 `[手动领取]`，或在活动 GUI 点击“领取物品”。
- 维护通知：支持维护时间和维护倒计时，两个输入框会互相同步。
- 探索/副本：用于多人探索或副本集合。

维护倒计时会在活动页实时显示。服务端会在 `60/30/15/10/5/4/3/2/1` 秒提醒，并在倒计时结束时执行 `/stop`。

活动发布时离线的玩家加入服务器后会收到一次活动聊天提示，不会反复刷屏。

## 服主管理

OP 等级 2 及以上可以使用服主管理页：

- 时间管理：白天、正午、夜晚、午夜。
- 天气管理：晴天、下雨、雷暴。
- 玩家管理：传送、游戏模式、临时飞行、恢复生命、补饥饿、清效果、熄灭、Kick、Ban、授予或撤销 OP。
- 清理管理：清理掉落物、经验球、箭矢类投射物、敌对生物。
- 重载配置：重新读取 `friendservermenu.json`。

普通玩家即使伪造客户端 payload 请求管理员 action 或活动组织 action，服务端也会拒绝。

## 坐标分享

公开坐标时，服务端会保存一个 5 分钟有效的临时坐标分享 ID。聊天栏绿色 `[点我传送]` 按钮只执行：

```text
/fsm_coord_tp <id>
```

真实坐标、维度、朝向和过期时间都由服务端保存，玩家客户端不会提交可信传送目标。

## 构建

```powershell
.\gradlew.bat build
```

构建完成后，产物位于：

```text
build/libs/friendservermenu-1.0.0.jar
build/libs/friendservermenu-1.0.0-sources.jar
```

## 项目结构

```text
src/main/java/      服务端与通用逻辑
src/client/java/    客户端 GUI、HUD、网络接收
src/main/resources/ fabric.mod.json、声音资源
```

## 不包含的功能

本 MOD 不实现领地、经济、商店、权限组、大厅、反作弊、数据库或网页后台。它的目标是给朋友服提供一个轻量、好用、由服务端校验的游戏内菜单。
